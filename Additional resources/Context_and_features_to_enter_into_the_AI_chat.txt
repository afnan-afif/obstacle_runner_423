# MASTER PROJECT CONTEXT: 3D OPENGL OBSTACLE RUNNER

## 1. PROJECT RULES & CONSTRAINTS
- This is a 3D Computer Graphics university project built in Python using legacy OpenGL and GLUT.
- NO Shaders and NO Textures are allowed. 
- We are strictly limited to the basic shapes provided in our template (`glutSolidCube`, `gluCylinder`, `gluSphere`) and basic transformations (`glTranslatef`, `glRotatef`, `glScalef`). 
- We are allowed to use `glEnable(GL_DEPTH_TEST)` for proper 3D rendering.
- The project must be contained entirely within a single `.py` file. 
- All code snippets generated by the AI from this point on MUST be converted directly into Python.

## 2. CORE GAMEPLAY (SUBWAY SURFERS CLONE)
- **Camera & Perspective:** A fixed perspective camera looking down the Z-axis.
- **Infinite Scrolling:** The player stays at a fixed Z-coordinate. The floor (a grid of GL_QUADS) and obstacles translate towards the camera along the Z-axis to create the illusion of forward movement.
- **Three-Lane Movement:** The player is a `glutSolidCube` that can smoothly interpolate between three fixed X-coordinates (e.g., Left = -100, Center = 0, Right = 100).
- **Player Actions:** - Jump: Time-based parabolic Y-axis translation.
  - Roll/Duck: Squashing the Y-axis using `glScalef`.
- **Obstacles:** Procedurally generated basic shapes that spawn far down the Z-axis and move toward the player.
- **Collisions:** Handled via 3D Axis-Aligned Bounding Box (AABB) mathematical checks.
- **HUD:** Real-time on-screen text showing Score and Lives.

## 3. NOVELTY FEATURES (For Bonus Marks)
To make the project unique, we are implementing three specific novelty mechanics:
1. **Dynamic Warp Speed:** As the score increases, the global game speed increases, and the camera's `fovY` dynamically widens to create a visual "warp" effect.
2. **Gravity Inversion:** A state where the player's Y-axis gravity flips, forcing them to run upside-down on a ceiling grid. Jump math must reverse accordingly.
3. **Morphing Obstacles:** Obstacles that change their shape (e.g., smoothly transitioning from a `gluCylinder` to a `gluSphere`) and update their bounding boxes as they cross a specific Z-axis threshold near the camera.
